# Лабораторная работа №2

## Тема
Изучение систем контроля версий на примере Git

## Цель работы
Познакомиться с основными возможностями системы контроля версий Git и сервиса GitHub, научиться создавать локальный и удаленный репозиторий, добавлять файлы в проект, выполнять commit, push и оформлять отчет по результатам работы.

## Задание
1. Создать публичный репозиторий `lab02`.
2. Сгенерировать token для GitHub с правами `repo`.
3. Ознакомиться с учебными материалами.
4. Выполнить инструкцию tutorial.
5. Подготовить отчет о выполнении лабораторной работы.

## Выполнение работы

В ходе выполнения лабораторной работы был создан публичный репозиторий `lab02` на GitHub.  
Для доступа к удаленному репозиторию был создан GitHub token с правами `repo`.  
После этого была выполнена настройка Git, указаны имя пользователя и электронная почта.

Далее был создан локальный репозиторий, выполнена его инициализация и подключение удаленного репозитория.  
В проект были добавлены основные файлы и каталоги, указанные в инструкции:

- `README.md`
- `.gitignore`
- `sources/print.cpp`
- `include/print.hpp`
- `examples/example1.cpp`
- `examples/example2.cpp`

После создания файлов изменения были добавлены в индекс Git, зафиксированы с помощью `commit` и отправлены в удаленный репозиторий с помощью `push`.

Также был изучен базовый порядок работы с Git:
- создание репозитория;
- добавление файлов под контроль версий;
- фиксация изменений;
- отправка изменений на GitHub.

## Основные команды, использованные в работе

```bash
export GITHUB_USERNAME=...
export GITHUB_EMAIL=...
export GITHUB_TOKEN=...

git config --global user.name "${GITHUB_USERNAME}"
git config --global user.email "${GITHUB_EMAIL}"

mkdir lab02
cd lab02
git init
git remote add origin https://github.com/${GITHUB_USERNAME}/lab02.git

touch README.md
git add README.md
git commit -m "added README.md"
git push -u origin master
